Lexia: IA jurídico-administrativa local para España

Un asistente RAG para consultar normativa, preparar borradores y reducir burocracia sin sacar datos sensibles de la organización. Cada respuesta muestra sus citas y enlaces a fuentes oficiales.

Probar la demo Ver requisitos

Consulta, redacta y verifica

Lexia prioriza fuentes oficiales, trazabilidad y revisión humana. El objetivo no es sustituir criterio jurídico, sino ahorrar tiempo en trabajo normativo repetitivo.

🔎

Consulta normativa

Pregunta en lenguaje natural. Lexia recupera artículos relevantes y responde citando cada fuente usada.

✍️

Borradores revisables

Genera informes, resoluciones, requerimientos, recursos, contratos o dictámenes con fundamentos citados y marcadores pendientes.

Fuentes trazables

La biblioteca permite explorar el corpus indexado y abrir la fuente oficial desde cada cita.

Datos bajo control

Pensada para sector público

La administración necesita IA práctica, responsable y auditada: menos fricción burocrática, más claridad para funcionarios, equipos jurídicos y ciudadanía.

Expedientes y trámites

Ayuda a localizar base normativa, requisitos, plazos y fundamentos aplicables antes de redactar o resolver.

Servicio al ciudadano

Permite construir asistentes que respondan con límites claros: si no hay fuente suficiente, la respuesta debe reconocerlo.

Integración con agentes

Incluye endpoints estables para que otros agentes consulten, recuperen fuentes y generen borradores citados.

Cómo funciona

RAG jurídico: primero busca, luego redacta, siempre con fuentes.

Preguntas

El profesional describe la duda, expediente o borrador que necesita preparar.

Recupera

El motor combina búsqueda léxica, embeddings y reranking para localizar fuentes relevantes.

Redacta

Un LLM local genera respuesta o borrador usando solo el contexto recuperado.

Verificas

La persona responsable revisa citas, enlaces oficiales y texto final antes de usarlo.

Por qué local importa

La privacidad no es un añadido: es una decisión de arquitectura.

Confidencialidad y RGPD

Consultas, documentos y datos de expedientes pueden permanecer dentro de la infraestructura de la organización.

Sin coste por token externo

El stack puede funcionar con modelos abiertos y servicios compatibles con OpenAI desplegados localmente.

El valor está en el flujo

El diferencial no es un modelo concreto, sino corpus fiable, recuperación medida, citas verificables e integración operativa.

Instalación local

El repositorio no incluye datos personales, corpus masivo ni modelos. Todo se genera o descarga en la infraestructura local de quien evalúa la herramienta.

npm run ingest npm run embed npm start

Requisitos resumidos: Node.js 18+, un servidor local compatible con OpenAI para chat y embeddings, un modelo instructivo en español, un modelo de embeddings multilingüe y, opcionalmente, un reranker. Consulta el README para la guía completa.

Lexia — IA jurídico-administrativa open source. Demo técnica; no sustituye la revisión profesional. Corpus BOE y embeddings son regenerables localmente; jurisprudencia CENDOJ está en roadmap.
